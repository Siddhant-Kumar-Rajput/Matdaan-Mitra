# MatDaan Mitra — मतदान मित्र
## Antigravity Agent Blueprint

You are a **senior full-stack developer** building "MatDaan Mitra" — a multilingual, interactive election education assistant for Indian voters. This is a hackathon submission for Google's Prompt Wars competition (Hack to Skill platform), evaluated on code quality, security, Google service integration, accessibility, and real-world usability.

---

## Project Vision

MatDaan Mitra (Your Voting Companion) helps Indian voters understand the election process through three personalised modules, accessible in multiple regional languages. The app must feel like a helpful human guide — not a government form.

**Core user flow:**
1. Language selection screen (first thing shown)
2. Smart onboarding (4 adaptive questions powered by Gemini)
3. Personalised dashboard with three module tiles
4. Each module opens a dedicated, context-aware experience

---

## Tech Stack

- **Frontend:** Vanilla HTML5, CSS3, JavaScript (ES6+) — no frameworks
- **AI:** Google Gemini API (`gemini-2.0-flash` model via REST)
- **Real-time data:** Gemini Google Search Grounding (`tools: [{ googleSearch: {} }]`) — fetches live election schedules, no scraping needed
- **Translation:** Google Translate API v2
- **Maps:** Google Maps Embed API (for booth locator)
- **Calendar:** Google Calendar API (for voting day reminder)
- **Fact-checking:** Google Fact Check Tools API
- **Storage:** `sessionStorage` only — zero persistent user data
- **Hosting:** Firebase Hosting (static)

> Do NOT use React, Vue, Angular, or any npm framework. Keep dependencies to zero. The judges will read the source code directly.

## Module Specifications

### Module 1 — Am I Election-Ready? (`checklist.html`)
- Gemini dynamically generates a personalised voter checklist from onboarding answers (state, constituency, first-time voter, EPIC card status)
- Each checklist item has: title, description, status toggle, and a "Why does this matter?" Gemini-powered tooltip
- **Booth locator:** Google Maps Embed API centred on user's constituency
- **Calendar export:** One-click "Add voting day to Google Calendar" with a 7-day reminder
- Checklist can be printed or saved as text

### Module 2 — How Elections Work (`timeline.html`)
- Interactive vertical timeline with 7 phases: Announcement → Nominations → Campaign → Voting Day → Counting → Results → Oath
- Each phase is a clickable card — clicking opens a Gemini-powered detail panel in the chosen language
- "What if?" scenario tab: user can ask edge-case questions (e.g. "What if EVM malfunctions?")
- Mini quiz at the end: 3 multiple-choice questions generated by Gemini based on the timeline content

### Module 3 — Myth vs Fact (`mythbuster.html`)
- User types any election claim they've heard
- Gemini classifies it as ✅ FACT / ❌ MYTH / ⚠ PARTIALLY TRUE
- Explanation shown in simple language, with a confidence note
- Google Fact Check Tools API cross-references the claim
- Session myth log shows all claims checked in this session
- "Share this fact-check" button generates a shareable text card

## Floating Gemini Assistant
- A persistent circular button (bottom-right, all pages) opens a slide-up chat drawer
- The assistant knows the user's language, state, and module context from sessionStorage
- System prompt includes the user's profile so answers are always personalised
- The drawer does not navigate away — it overlays the current page

## Coding Standards

### General
- All JS functions must have JSDoc comments
- Use `const` and `let` — never `var`
- Maximum function length: 30 lines. Extract helpers if longer
- Meaningful variable names — no single-letter variables except loop counters
- All magic numbers must be named constants in `config.js`
- Every async function must have a try/catch with a user-visible error state

### CSS
- Use CSS custom properties (variables) for all colors, spacing, and font sizes
- Mobile-first responsive design — test at 360px, 768px, 1024px
- No inline styles in HTML — all styles in CSS files
- BEM naming convention for class names

### Accessibility (WCAG 2.1 AA minimum)
- Every interactive element must have an `aria-label` or visible label
- Color contrast ratio minimum 4.5:1
- All images must have `alt` text
- Keyboard navigable — tab order must be logical
- Focus states must be visible
- Screen reader tested with landmark regions (`<main>`, `<nav>`, `<header>`)

### Performance
- No blocking scripts in `<head>` — use `defer` or `async`
- Images must use `loading="lazy"`
- Gemini API calls must show a loading skeleton, not a spinner blocking the whole page
- Debounce any input that triggers an API call (300ms minimum)

## Security Requirements

### API Key Handling
- API keys are NEVER hardcoded in source files
- Load from `config.js` which reads from environment variables set in Firebase Hosting config
- Add `config.js` to `.gitignore`
- Provide a `config.example.js` with placeholder values

### Input Sanitisation
- ALL user input must be sanitised before being passed to Gemini
- Use the `sanitiseInput()` function in `gemini.js` — strip HTML tags, limit to 500 characters
- Validate that input is non-empty and contains meaningful text before making an API call

### Prompt Injection Guard
- The Gemini system prompt in every module must include the guardrail block (see `responsible-ai` skill)
- Log any rejected queries to `sessionStorage` for the session myth log

### Content Security Policy
- Add CSP headers in `firebase.json`:
  - `default-src 'self'`
  - `connect-src` only whitelisted Google API domains
  - `script-src 'self'` — no inline scripts
  - `style-src 'self' 'unsafe-inline'` (required for dynamic CSS variables only)

## Responsible AI Guardrail

Every Gemini API call in this project must use this system prompt prefix. Never remove it:

```
You are MatDaan Mitra, a helpful, neutral election education assistant for Indian voters.

STRICT RULES — you must follow these in every response:
1. You are strictly NON-PARTISAN. Never say anything that favours or criticises any political party, candidate, leader, or ideology.
2. If the user's query is politically biased, inflammatory, about specific politicians, or asks you to compare parties, respond ONLY with: "I can only help with election process information, not political opinions."
3. Never generate content that could be used as election propaganda.
4. If the query is in [LANGUAGE], respond entirely in [LANGUAGE].
5. Keep all explanations simple — assume the user has no prior knowledge of election law.
6. When uncertain about a specific legal detail, say so and direct the user to the Election Commission of India website (eci.gov.in).
```

Replace `[LANGUAGE]` dynamically with the user's chosen language from sessionStorage.

## Error Handling Standards

Every API call must follow this pattern:
```javascript
async function callGemini(prompt) {
  showLoadingSkeleton();
  try {
    const response = await geminiAPI(prompt);
    hideLoadingSkeleton();
    return response;
  } catch (error) {
    hideLoadingSkeleton();
    showErrorState('Could not get a response. Please check your connection and try again.');
    logError(error); // sessionStorage only, never external
  }
}
```

Never show raw error objects or stack traces to the user.
